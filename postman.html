Documentación de Pruebas en Postman

Este script se utiliza para realizar pruebas automatizadas en una API mediante Postman. A continuación, se explican las distintas secciones del código y su funcionalidad.

1. Validación del Código de Estado

pm.test("Status code is 200", function () { pm.response.to.have.status(200); });

Esta prueba verifica que la respuesta de la API tenga un código de estado 200, indicando que la solicitud fue exitosa.

2. Tiempo de Respuesta

pm.test("API responde en tiempo esperado 800ms", function () { pm.expect(pm.response.responseTime).to.be.below(800); });

Esta prueba valida que la respuesta de la API se reciba en menos de 800 milisegundos, asegurando un rendimiento adecuado.

3. Validación de Campos Requeridos

pm.test("Respuesta incluye campos requeridos", function () { pm.expect(pm.response.text()).to.include("username"); pm.expect(pm.response.text()).to.include("address"); pm.expect(pm.response.text()).to.include("city"); });

Esta prueba verifica que la respuesta de la API contenga los campos "username", "address" y "city", que son requeridos para la correcta funcionalidad.

4. Validación de Otros Campos Requeridos

pm.test("Respuesta incluye campos requeridos", function () { pm.expect(pm.response.text()).to.include("albumId"); pm.expect(pm.response.text()).to.include("title"); pm.expect(pm.response.text()).to.include("url"); });

Esta prueba asegura que la respuesta también contenga los campos "albumId", "title" y "url", los cuales son esenciales para el funcionamiento correcto de la aplicación.

5. Validación de Más Campos Requeridos

pm.test("Respuesta incluye campos requeridos", function () { pm.expect(pm.response.text()).to.include("name"); pm.expect(pm.response.text()).to.include("email"); pm.expect(pm.response.text()).to.include("body"); });

Esta prueba final verifica que la respuesta contenga los campos "name", "email" y "body", asegurando que todos los datos necesarios están presentes.
